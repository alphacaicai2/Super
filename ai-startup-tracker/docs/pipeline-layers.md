# 分层流水线：召回 → 正文精判 → 结构化抽取 → 人工复核

设计原则：**召回尽量别漏，抽取尽量准**。不依赖「只筛标题」、也不一上来就微调，用分层把「标题党/隐晦标题」先捞出来，再在正文里精判、最后做结构化抽取与人工复核。

策略参考：[ChatGPT - 信息筛选与抽取策略](https://chatgpt.com/s/t_699963a62cc08191afe3703913c01284)（标题粗筛 + 正文精判 + 结构化抽取 + 人工复核；先打分/通用模型，再沉淀数据，后考虑微调）。

---

## 容易踩的坑（中国自媒体）

标题常见「标题党/隐晦」：*他又出手了*、*这一夜，行业变了*、*某大厂悄悄上线*、*融资不是重点，重点是…*  
**只看标题关键词会漏很多**，标题只能做低成本预筛，不能做最终判断。正文前 200–500 字通常才会写「谁、做了什么、金额多少」，比标题更可靠，因此召回阶段用「标题 + 正文前段」一起筛。

---

## 四层结构

| 层级 | 名称 | 作用 | 实现 |
|------|------|------|------|
| 1 | **召回（标题粗筛）** | 从一堆文章里捞出「可能相关」的，宁可多捞、别漏 | `preprocess`：关键词表 / minimal / 或 `FUNDING_PREFILTER=none` 全送下一层 |
| 2 | **正文精判** | 看正文是否真的含有可抽取的投融资/产品发布事实，过滤标题党与无关文 | `pipeline/classify.py`：LLM 二分类 YES/NO（`BODY_CLASSIFY=true` 时开启） |
| 3 | **结构化抽取** | 仅对精判为 YES 的做 LLM 抽取：公司、轮次、金额、投资方等 | `pipeline/extract.py` + `write_airtable.py` |
| 4 | **人工复核** | 低置信度、缺字段、Undisclosed 等标 needs_review，在 Airtable 待复核视图里改 | `write_airtable.should_review` + 待复核视图 |

---

## 配置

- **召回**：`FUNDING_PREFILTER=keyword|minimal|none`（keyword=全表，minimal=少量宽泛词，none=不预筛全部进正文精判）
- **正文精判**：`BODY_CLASSIFY=true|false`（默认 true，关掉则召回后直接进结构化抽取）
- 若关掉正文精判（`BODY_CLASSIFY=false`），则退化为「召回 → 结构化抽取 → 人工复核」三层。

---

## 统计与 Telegram

- **召回跳过**：标题粗筛未命中，未进入正文精判（原「关键词未命中跳过」）
- **正文精判跳过**：精判为 NO，未进入结构化抽取
- 摘要中会分别展示「召回跳过」与「正文精判跳过」，便于调参与观察漏斗。

## 与策略文档的对齐

| 策略建议 | 当前实现 |
|----------|----------|
| 召回：标题 + 正文前段 + 宁可多捞 | 召回用「标题 + 正文前 2000 字」关键词 / minimal / none；keyword 表含融资+产品类（上线、发布、beta 等） |
| 正文精判：是否含可抽取事实，过滤标题党 | `BODY_CLASSIFY`：LLM 二分类 YES/NO（正文前 4000 字） |
| 结构化抽取：JSON schema、证据句、置信度 | `extract_funding` 出 schema；FundingRounds 存 `evidence_text`、`confidence`；低置信度/缺字段标 needs_review |
| 人工复核、复核数据沉淀 | 待复核视图；可后续用复核结果做训练集 |
| 先不做：微调、只靠标题、无 evidence 的黑盒 | 未做微调；召回看标题+正文前段；抽取带 evidence_text |

**后续可做（未实现）**：召回分 `recall_score`（标题+前段+来源+历史命中率）与阈值；正文精判改为多标签事件分类（融资财报 / 产品发布/上线 / 并购/合作 / 无关）；产品发布事件的结构化抽取与表结构。
